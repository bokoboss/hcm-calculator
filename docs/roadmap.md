# Roadmap

The following is a planning inventory, not a commitment to broaden the
qualified HCM scope in a maintenance release.

## Maintenance candidates

- Correct verified implementation defects without silently changing qualified
  numerical behavior.
- Refine the single-page worksheet UX, terminology, help text, accessibility,
  and localization.
- Improve packaging and local distribution for supported Python versions.
- Improve documentation, provenance display, export ergonomics, performance,
  and automated regression coverage.

## Methodology expansion candidates

| Candidate | Required path before product support |
| --- | --- |
| Ramp influence areas | Separate reference audit, engine phase, and integration/release phase. |
| Merge/diverge | Separate reference audit, engine phase, and integration/release phase. |
| Weaving | Separate reference audit, engine phase, and integration/release phase. |
| Freeway facilities | Separate reference audit, engine phase, and integration/release phase. |
| Additional HCM domains | Separate reference audit, engine phase, and integration/release phase. |

Any methodology candidate must define scope, authoritative references,
validation fixtures, expected outputs, audit fields, persistence behavior,
exports, and release qualification before implementation starts.
